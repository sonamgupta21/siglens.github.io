The popular programming language Go includes a built-in logging package, however it lacks crucial components needed for production-level logging.


The machine-readable JSON format is used to structure the message. It has a level that indicates how serious the message is. Structured Logging

The process of structured logging entails formatting log messages in a structured format, such as JSON or key-value pairs. This organized method makes it simpler to programmatically interpret and analyze logs, facilitating improved monitoring and debugging.

The slog.Record type can be used to construct structured log entries with the log/slog package. This type makes it simpler to provide context to your logs by allowing you to include key-value pairs in your log entries. 


log := slog.New(os.Stderr, slog.JSONFormat(), slog.InfoLevel)
log.Record(slog.Fields{
"user":   "alice",
"action": "login",
"status": "success",
}).Info("User login")

Zerolog


Zerolog is an organized and high-performance Go logging package. It is renowned for its extraordinary speed, ability to annotate log records with context, and support for structured logging. Let's look at its key characteristics and discover how to utilize it.

Features of Zerolog

  • Performance:
    1. Fastest in Class: Zerolog accomplishes this by thoroughly streamlining its internal implementation, making it among the fastest Go logging frameworks available.
    2. Low Allocation: A key component of Zerolog's architecture is memory reduction by unnecessary allocations, which is crucial for high-performance applications.
  • Structured logging: Zerolog provides structured logging in both JSON and CBOR (Concise Binary Object Representation) formats.
  • Contextual Logging: You can add fields or contextual characteristics like user IDs, request IDs, or other custom data relevant to your application's context to log records in Zerolog.
  • Ease of Use:
    1. API simplification: Developers only need to configure the logger, set output options, and define log levels.
    2. Chaining Support: Zerolog's support for method chaining simplifies the work of writing complex log messages with several fields.
  • Active Community: Zerolog has a growing and active user and contribution base. It indicates that you have access to a wealth of resources, discussions, and third-party integrations to help you make effective use of the library.
  • Output Adapters: Zerolog allows you to customize log output by attaching different output adapters. Consequently, logs can be sent to a number of destinations, including files, the normal output, and other services like Logstash and Elasticsearch.
  • Log Levels: The library supports various log levels so you may control the amount of verbosity in your logs. Logs can be filtered using these levels to provide you only the data required for monitoring or debugging.

Installation 

go get github.com/rs/zerolog

How to Use Zerolog?


Basic Logging: 

package main

import (
  "os"
  "github.com/rs/zerolog"
)
                                        
func main() {
  // Create a logger instance with the console as the output.
  logger := zerolog.New(os.Stdout)
                                        
  // Set the log level to Debug for verbose logging.
  logger = logger.Level(zerolog.DebugLevel)
                                        
  // Log a message with a field.
  logger.Debug().Str("key", "value").Msg("Debug message")
                                        
  // Log an error with a stack trace.
  logger.Error().Err(someError).Stack().Msg("An error occurred")
}

Structured Logging: 

logger.Info().
Str("event", "user_login").
Str("user_id", "123").
Msg("User logged in")

Log Formats:

Zerolog supports multiple log formats, including JSON and CBOR (Concise Binary Object Representation).

Adding Contextual Properties:

logger := zerolog.New(os.Stdout).With().Str("app_name", "MyApp").Logger()

logger.Info().Msg("Application started")

Zerolog stands out for Go developers, offering top-tier logging with minimal performance impact. Its support for multiple log formats, low overhead, and simplicity in adding context make it ideal for diverse project sizes, maintaining application performance while efficiently capturing and analyzing log data—from microservices to large-scale applications.


Zap

Zap is a feature-rich, extremely effective Go logging library with a number of customization choices. It is widely appreciated for its agility and speed. In this blog, we'll go over some of its key features and provide code samples to demonstrate how to utilize it.

Features of Zap

  • High Performance Logging
  • Structured Logging
  • Logger Configuration:Zap offers a range of configuration options for loggers, including the ability to define log output destinations, select output formats, and set log levels.
  • Field Attachment: Zap enables you to attach key-value pairs, or fields, to log entries. This feature is very useful for adding context to your log records.
  • SugaredLogger: SugaredLogger provides a more user-friendly and readable API for humans.
  • Initialization of the Global Logger: Zap requires that the global logger be manually initialized in order to ensure that the settings are accurate and consistent across your application. This is to guarantee that your application's logging runs reliably and securely.

Installation 

go get -u go.uber.org/zap

Basic logger with Zap 

package main

import (
    "go.uber.org/zap"
)
                                        
func main() {
    // Create a logger instance.
    logger, _ := zap.NewProduction()
                                        
    // Log a message.
    logger.Info("Informational message")
                                        
    // Log an error.
    logger.Error("An error occurred", zap.Error(someError))
}

SugaredLogger 

package main

import (
    "go.uber.org/zap"
)
                                        
func main() {
    // Create a SugaredLogger.
    logger, _ := zap.NewProduction()
    sugar := logger.Sugar()
                                        
    // Log a message using the SugaredLogger.
    sugar.Infow("Informational message", "key", "value")
}

Initializing the global logger 

package main

import (
    "go.uber.org/zap"
)
                                        
func main() {
    // Initialize the global logger.
    logger, _ := zap.NewProduction()
    zap.ReplaceGlobals(logger)
                                        
    // Log a message using the global logger.
    zap.L().Info("Global logger message")
}

In the Go environment, Zap is a premium logging library with excellent speed and sophisticated customisation capabilities. It is a fantastic choice for developers who require efficient logging in their applications, whether they are large-scale systems or microservices.


Slog

Go 1.21 brought with it a potent logging library called Slog. Its goal is to supply leveled and structured logging solutions for Go programs. Slog is a structured logging system that enables you to efficiently create, manage, and assess log data.

Features of Slog

  • Structured Logging
  • Leveled Logging:To distinguish between log messages, you can use different log levels such as Info, Debug, Warn, and Error. For example:
    • logger.Info(slog.String("key", "value"), "Informational message")
logger.Warn(slog.Error(someWarning), "Warning message")
logger.Error(slog.Error(someError), "An error occurred")
  • Logging Process
    The logging process is divided into three sections by Slog:
    Frontend Logger: The Logger serves as your interface for creating log entries. It is employed to add structured data to log entries and select a log level.
    Record Type: Records are created by the Logger. They store the log data and are designed to be easily manipulated.
    Handler Backend: Handlers are responsible for processing log records. By allowing you to choose various backend handlers to handle log entries in various ways—for example, sending logs to external services or files—Slog offers versatility.
  • Custom Log Formats and Third-Party Backends: This suggests that you may set up Slog to output logs in a manner that suits your needs and connect it to services like Elasticsearch, Logstash, and others. For example: 
    • import (
    "github.com/go-kratos/slog"
    "github.com/go-kratos/slog/file"
)
                                        
func main() {
    // Create a file handler to write logs to a file.
    handler, err := file.New(file.WithPath("app.log"))
    if err != nil {
    panic(err)
}
                                        
    // Create a logger with the file handler.
    logger := slog.New(slog.WithOutput(handler))
                                        
    // Log a message with structured data.
    logger.Info(slog.String("key", "value"), "Informational message")
}

Installation 

go get github.com/go-kratos/slog

Basic logging with Slog: 

package main

import (
    "fmt"
    "github.com/go-kratos/slog"
)
                                        
func main() {
    // Create a new logger.
    logger := slog.New()
                                        
    // Log an informational message.
    logger.Info("This is an informational message.")
                                        
    // Log a warning message with additional context.
    logger.Warn(slog.String("key", "value"), "This is a warning message with additional context.")
                                        
    // Log an error message and associate an error.
    err := fmt.Errorf("An error occurred")
    logger.Error(slog.Error(err), "This is an error message.")
                                        
    // Log a debug message (for completeness; debug logs may not always be shown).
    logger.Debug("This is a debug message. You might not see it by default.")
                                        
    // Log a custom log level (Slog supports various log levels).
    logger.Log(slog.Level(slog.LevelInfo+1), "This is a custom log level.")
                                        
    fmt.Println("Logging completed.")
}

Structured logs with Slog: 

package main

    "github.com/go-kratos/slog"
)

    // Create a logger instance.
    logger := slog.New()

    logger.Info(slog.String("key", "value"), "Informational message")

    logger.Error(slog.Error(someError), "An error occurred")
}

Benefits from Slog's structured logging approach, support for custom log formats, and integration with third-party backends can be extended to small utilities and large-scale systems. Whether you're logging errors, alerts, or helpful remarks, Slog enables you to efficiently handle and examine log data in your Go applications.


Logrus

In the Go environment, Logrus is an established and extensively utilized logging library. Its status as one of the early pioneers of structured logging in the Go community makes it a suitable choice for developers that require a flexible and organized approach to logging. In order to demonstrate how to utilize Logrus, we will go over its key features, installation instructions, and code samples in this blog.

Features of Logrus

  • Structured Logging
  • Various Output Formats:Logrus works with a number of output types, including Text, JSON, and Logfmt.
  • Contextual Logging
  • Log Levels

Installation 

go get github.com/sirupsen/logrus

Basic Logging 

package main

import (
    "github.com/sirupsen/logrus"
)
                                        
func main() {
    // Create a new logger instance.
    logger := logrus.New()
                                        
    // Set the log level to control verbosity.
    logger.SetLevel(logrus.DebugLevel)
                                        
    // Log an informational message.
    logger.Info("This is an informational message.")
                                        
    // Log a warning message with additional context.
    logger.WithFields(logrus.Fields{
        "key": "value",
    }).Warn("This is a warning message with additional context.")
                                        
    // Log an error message.
    logger.Error("An error occurred.")
}

Structured Logging 

package main

import (
    "github.com/sirupsen/logrus"
)

func main() {
    // Create a new logger instance.
    logger := logrus.New()

    // Set the log level to control verbosity.
    logger.SetLevel(logrus.InfoLevel)

    // Log a message with structured data using WithFields.
    logger.WithFields(logrus.Fields{
        "user":   "john_doe",
        "action": "login",
    }).Info("User logged in")

    // Log an error with additional structured fields.
    logger.WithFields(logrus.Fields{
        "error_code":    500,
        "error_message": "Internal Server Error",
    }).Error("An error occurred")

    // Log a warning message with more structured fields.
    logger.WithFields(logrus.Fields{
        "process_id": 12345,
        "task":       "data_processing",
    }).Warn("A warning with additional context")

    // Log a message without structured fields.
    logger.Info("This is a basic log entry without structured fields.")
}

The ability to add context with WithFields allows you to provide additional information in your log entries.

Regardless of whether you're creating a command-line tool, web service, or distributed application, Logrus provides the flexibility and capabilities you need to effectively gather and analyze log data.


Log15

A flexible logging package for Go called Log15 aims to give logs that are both machine- and human-readable. Its user-friendly API and pre-configured logger make it accessible to developers with different degrees of expertise. In this blog post, we'll go over how to install Log15 and some of its key features that make it a fantastic choice for Go program logging.

Features of Log15

  • Log Levels
  • Human-Readable and Machine-Readable Logs:Log15 offers both human-readable log style that is easy for developers to read and understand and machine-readable JSON logging for efficient log analysis and parsing.
  • Pre-Configured Logger and Simple API: With Log15's pre-configured logger, you can start using it right away and save a ton of setup time. Its clear and easy-to-use API makes it simple to create log entries with minimal coding.
  • Various Log Formatting and Handling Options: Depending on your needs, log output can be set up in a number of formats, including plain text, JSON, and other prepared formats. Its support for numerous handlers allows you to deliver logs to a range of destinations, such as log files, external services, and custom endpoints.

Installation

To use Log15 in your Go project, follow these installation steps:

Open your terminal or command prompt.
Run the following command to install Log15:

    // Log a warning message.

    // Log an error message.

Structured Logging 

package main

import (
    "gopkg.in/inconshreveable/log15.v2"

Log15 provides the capabilities and flexibility required for developers working on large-scale applications, web services, microservices, command-line tools, and other applications to efficiently gather, handle, and store log data.


Logf

A straightforward Go logging package called Logf is intended to produce structured logs in the Logfmt format. It offers a lightweight, simple API with some basic customization options, making it a viable choice for developers searching for simplicity and structure in their logs. In this blog post, we will go over the installation procedure, key features, and code samples for using Logf.

Features of Logf

  • Logfmt Format:Logf focuses on generating Logfmt format logs, a structured logging format that is both machine- and human-readable. Its utilization of key-value pairs allows for easy parsing and analysis of log entries. Here's an example of Logfmt log entry:
    • event=user_login user=john_doe
  • Minimal API
  • Structured Logging
  • Lightweight and Efficient
  • Easy Integration

Installation

Open your terminal or command prompt.
Run the following command to install Logf: 

go get github.com/go-logfmt/logfmt

    "os"

    logger := logfmt.NewLogger(os.Stdout)

    logger.Log("event", "user_login", "user", "john_doe")

    // Log another event.
    logger.Log("event", "file_upload", "user", "jane_smith")

Structured Logging

import (

For Go structured logging, Logf is a practical and reasonably priced solution. It is excellent in providing logs in the Logfmt format, with an emphasis on striking a balance between readability and structured data.

Logr

Logr provides an abstraction layer that allows developers to log messages without tying their code to a particular logging implementation. In fact, it provides a uniform method for logging messages throughout the program by defining a Logger class with a simple API.

Features of Logr

  • Logger Type with Simple API
  • Abstraction Layer:Logr serves as an abstraction, allowing developers to write log messages without concern for the underlying logging mechanism. It is simple to switch between several logging implementations because to this separation.
  • Dependency on LogSink Interface:Logr's logging capabilities are dependent on types that have LogSink interface implementations. This interface encourages adaptability and customization by defining the necessary techniques for handling log entries.

Installation 

go get -u github.com/go-logr/logr

Basic Logging 

package main

import (
    "github.com/go-logr/logr"
	"log"
)

func main() {
	logger := logr.NewLogger() // Create a new Logger instance

	// Basic logging
	logger.Info("This is an info log message")
	logger.Error("This is an error log message")
}

Structured Logging 

package main

import (
    "github.com/go-logr/logr"
	"log"
)

func main() {
	logger := logr.NewLogger() // Create a new Logger instance

	// Structured logging with fields
	logger.WithValues("key1", "value1").
		WithValues("key2", 42).
		Info("Structured log message with fields")
}

Logr, a robust Go logging abstraction, simplifies log management by utilizing the LogSink interface, allowing seamless swapping of logging backends. Its independence from implementations streamlines code, enhancing readability and maintainability. Within the Go ecosystem, Logr strikes a balance between simplicity and extensibility, empowering developers to focus on application logic while ensuring flexible and efficient logging.


Benchmark results of different logging libraries of go

Benchmarking various Go logging libraries provided valuable insights into their performance differences. Zap stood out with its zero-allocation approach, logging significantly faster than Logrus and Apex/log. While Logrus and Apex/log balanced features with moderate speed, the standard library's simplicity resulted in slower performance. Recent entries like log/slog introduced compatibility among logging frameworks, allowing seamless transitions without code changes. Zerolog emphasized speed and simplicity with JSON-based structured logging, contrasting Zap's unmatched performance owing to its zero-allocation architecture. Slog's adaptability as a mediator between diverse logging technologies showcased its significance, simplifying integration within Go programs. Consider these benchmarks and project needs for an optimal logging framework choice.
